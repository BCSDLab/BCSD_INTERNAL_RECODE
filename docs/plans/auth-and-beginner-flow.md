# 인증 플로우 개선 + 가입 후 초기 플로우

## 현재 상태

### 로그인
- Google OAuth로 로그인
- 가입된 계정 → 로그인 완료
- 미가입 계정 → 401 `"member not found, registration required"` → 회원가입 페이지로 리다이렉트

### 회원가입
- 4단계 위저드: Google 인증 → 정보 입력(이름, 학과, 학번, 전화번호) → 학교 이메일 인증 → 트랙 선택
- 로그인에서 리다이렉트 시 Google 인증 단계 스킵
- 하단에 "이미 계정이 있으신가요? 로그인" 링크

---

## 인증 플로우 개선

### 로그인 → 회원가입 리다이렉트 UX 개선
- 현재: 미가입 시 "로그인에 실패했습니다" 에러가 잠깐 보임
- 개선: 에러 대신 "가입이 필요합니다" 안내 후 자연스럽게 회원가입으로 이동
- 로그인 페이지 하단에 "처음이신가요? 회원가입" 링크 추가

### 회원가입 경고 배너
- 문제: 아무 Google 계정으로 가입하는 경우가 있음
- 해결: 회원가입 위저드 상단에 경고 배너 표시
  - "BCSD에서 계속 사용할 Google 계정으로 가입해주세요."
  - "다른 계정으로 가입하면 나중에 변경이 어렵습니다."
- 기존 회원은 회원가입 페이지를 거치지 않으므로 경고를 안 봄

### 이미 가입된 계정으로 회원가입 시도
- "이미 가입된 계정입니다" 안내 → 로그인 페이지로 유도

### 회원가입 시 재학 정보 수집
- 기존: 이름, 학과, 학번, 전화번호
- 추가: 학년 (1~4학년, 대학원 등)
- 학년 정보를 기반으로 가입 후 분기 (아래 "가입 후 분기" 참고)

### 학교 이메일 기반 다중 Google 계정 연결 (장기)
- 회원의 고유 키를 학교 이메일로 변경
- 하나의 회원에 여러 Google 계정 연결 가능 (다대일)
- 다른 Google 계정으로 로그인해도 학교 이메일로 기존 회원과 매칭
- 이렇게 되면 경고 배너의 중요도가 낮아짐 (잘못된 계정으로 가입해도 나중에 연결 가능)

## 가입 후 분기

가입 완료 후 학년에 따라 다른 경로로 유도:

```
가입 완료
  │
  ├─ 1, 2학년 (또는 관리자가 지정한 기준 이하)
  │    → 비기너 지원 페이지로 유도
  │
  └─ 3, 4학년 이상 (또는 관리자가 지정한 기준 이상)
       → 즉시 전환 신청 페이지로 유도
       → "경력이 있으시다면 비기너 과정 없이 바로 레귤러로 합류할 수 있습니다"
```

- 학년 기준은 관리자가 설정 가능해야 함 (예: "3학년 이상은 즉시 전환 유도")
- 유도일 뿐 강제는 아님. 3학년도 비기너로 지원할 수 있고, 1학년도 즉시 전환을 신청할 수 있음 (단, 승인은 관리자 재량)

## 비기너 지원 플로우

### 전체 흐름

```
bcsdlab.com "지원하러 가기"
  │  (모집기간 아니면 "현재 모집 기간이 아닙니다" 표시)
  ▼
internal 로그인
  ├─ 기존 회원 → 로그인 완료
  └─ 미가입 → "가입이 필요합니다" → 회원가입 (⚠️ 경고 배너)
       → 가입 완료 → 학년 기반 분기
  ▼
비기너 지원 페이지
  │  조건 분기:
  │    - 이미 활동 중인 회원 → "이미 활동 중인 회원입니다"
  │    - 이미 지원서 제출 → 지원 상태 페이지로 이동
  │    - 모집 기간 아님 → "현재 모집 기간이 아닙니다"
  │    - 지원 가능 → 폼 표시
  ▼
지원 완료 → 지원 상태 페이지
  │  상태: 납부 대기 / 승인
  │  납부 전이라면 지원 취소 가능
  │  납부 후에는 취소 불가
  ▼
[관리자/부회장] 납부 확인 → 승인
  ▼
승인 시:
  │  비기너로 상태 변경
  │  슬랙 온보딩 안내
  ▼
비기너 활동 시작
```

### 지원 폼
- 관리자(회장/부회장 또는 레귤러 이상 권한)가 폼 질문을 편집 가능
  - 구글폼처럼 질문 추가/삭제/순서 변경/필수 여부 설정
  - 질문 타입: 단답형, 장문형, 객관식, 체크박스 등
- 기본 고정 필드 (수정 불가):
  - 이름, 이메일, 학교 이메일 — 자동 채움
  - 희망 트랙 — 자동 채움, 수정 가능

### 지원 상태
- 납부 대기: "회비 납부가 확인되면 승인됩니다." + 납부 안내 정보
  - 납부 전: "지원 취소" 버튼 표시
  - 납부 후: 취소 불가
- 승인: "BCSD 비기너로 승인되었습니다!" + 슬랙 온보딩 안내

### 관리자 지원자 관리
- 지원자 목록: 이름, 이메일, 희망 트랙, 지원일, 상태
- 납부 확인 → 승인
- 일괄 승인 가능
- 지원 폼 질문 편집 기능

## 즉시 전환 신청 플로우

3, 4학년 이상 또는 경력자가 비기너 과정 없이 바로 레귤러로 합류하는 경로.

- 즉시 전환자는 **가입비 없음**
- 전환 승인 시 곧바로 **레귤러 회비** 납부 대상이 됨

### 전체 흐름

```
가입 완료 (3, 4학년 이상)
  │  "비기너 과정 없이 바로 합류하시겠습니까?"
  │  [즉시 전환 신청] [비기너로 지원]
  ▼
즉시 전환 신청 페이지
  │  신청 폼 (관리자가 편집 가능, 비기너 지원 폼과 동일한 시스템)
  │  - 경력/포트폴리오 관련 질문 등
  │  - 가입비 없음 안내
  ▼
신청 완료 → 신청 상태 페이지
  │  상태: 심사 중 / 승인
  │  (가입비가 없으므로 "납부 대기" 없음)
  ▼
[관리자] 심사 → 승인
  ▼
승인 시:
  │  레귤러로 상태 변경
  │  레귤러 회비 납부 안내
  │  슬랙 온보딩 안내
  ▼
레귤러 활동 시작
```

### 비기너 지원 폼과의 관계
- 즉시 전환 신청 폼도 동일한 "편집 가능 폼" 시스템을 사용
- 관리자가 비기너 지원 폼과 즉시 전환 신청 폼을 각각 별도로 편집
- 폼 시스템은 하나, 폼 인스턴스가 여러 개인 구조

## 슬랙 온보딩
- Slack 무료 플랜이라 자동 초대 API 사용 불가
- 승인 시 슬랙 초대 링크를 상태 페이지에 표시
- 추후 유료 플랜 전환 시 자동 초대로 업그레이드 가능

## 모집 기간 관리
- 관리자가 모집 시작/종료일 설정
- bcsdlab.com과 지원 페이지에서 모집 기간 여부에 따라 분기
- 모집 기간 아닐 때 직접 URL 접근해도 "모집 기간이 아닙니다" 표시

## 권한 체계
- SpiceDB (Zanzibar 기반 권한 관리) 사용
- 프론트에서는 권한 체크 결과만 사용 (특정 액션 가능 여부)
- 권한이 필요한 기능 예시:
  - 지원자 관리, 폼 편집, 모집 기간 설정, 멤버 승인
  - 학년 기준 등 관리자 설정
- 역할/권한의 정의와 관계는 SpiceDB 스키마에서 관리 (백엔드)
